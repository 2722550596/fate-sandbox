# 时间感知模块

时间经过必须改变场景、身体和 NPC 行动窗口。

## 时间锚定

- 早晨、午后、傍晚、深夜的光线、人流、温度、声音密度都不同。
- 上课、午休、放学、通勤、末班车会改变地点可用性和 NPC 出现概率。
- 十分钟以上的等待、移动、休息、上课、通勤，都要体现至少一个时间痕迹。
- 用户要求等待、休息、睡眠、守夜或过夜时，时间经过是 canonical state 变化；不能只更新记忆/状态却漏掉 clock。
- 只有地点真的改变时才用 `move-location`；原地等待/过夜不要伪造地点移动。
- 数小时前、数天前、数周前的事不能写成「刚才」或「昨天」。

## 工具边界

本技能只判断时间经过应如何影响叙事，不决定具体工具路由。等待、休息、睡眠、守夜或过夜需要推进 clock 时，按工具策略模块用 `commit_turn` 的顶层 time 裁决提交。

## 长时间跳过

半天以上的跳时不要流水账。用 2-4 个镜头压缩：

1. 开始时的姿势、天气、人群或目的。
2. 中途的重复、磨损、疲劳、饥饿、光线变化或灵性残留。
3. 结束时的新局面、异常信号、NPC 行动结果或可选路径。

长时间跳过时，敌方、同伴、普通社会也在行动。不要把世界冻结等玩家。

## LOTM 地点生活节奏

- 廷根：中等工业城市节奏。清晨煤烟和工厂汽笛、上午值夜人分局交接、午间街道上报纸叫卖声、午后技术学校铃声，傍晚煤气灯点燃时的光线变化，入夜后街巷迅速安静。
- 贝克兰德：大都会昼夜不歇。白天商业区马车与有轨电车频繁，西区工厂区蒸汽轰鸣不绝，东区贫民窟巷道拥挤嘈杂。夜间上流社区灯光明亮但街道空荡，东区入夜后成为不同规则的地域。雾霾改变能见度，也改变什么行为能被看见。
- 拜亚姆：港口城市时间由潮汐与季风定。日出后码头开始装卸，午后因热带高温多数人进入室内或阴影，傍晚贸易风起时港口最活跃，深夜只有酒馆和教会岗哨还有灯光。海盗出没时间集中在黄昏后至前半夜。
- 康斯坦特：因蒂斯共和国革命情绪浓厚的大学城。白天咖啡馆和书店人流密集，夜间路灯稀疏处常有秘密集会。教堂钟声既是时间标尺，也是教会巡查的隐形闹钟——错过整点祷告辞可能被发现异常身份。
